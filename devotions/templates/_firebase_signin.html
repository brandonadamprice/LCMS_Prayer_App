{# Progressive enhancement for the "Sign in/up with Google" button. Include this in any page that has an button. The anchor keeps working exactly as before when JavaScript is unavailable; this script intercepts the click and signs in through Firebase Authentication (popup) instead, posting the ID token to the /auth/firebase session bridge. This is the flow that also works inside native app shells, where Google blocks its OAuth pages in embedded webviews. Failure policy: any unexpected failure (config fetch, SDK load, blocked popup, bridge error) falls back to the legacy /login/google redirect, so the worst case is identical to the pre-Firebase behavior. A deliberately closed popup just resets the button, and a 409 from the bridge carries a real user-facing message, so neither falls through to the legacy flow. Native shell (Capacitor): the sign-in goes through the OS-level Google flow via the FirebaseAuthentication plugin instead of the web popup, and failures NEVER fall back to /login/google -- Google refuses its OAuth pages inside webviews (disallowed_useragent), which is the whole reason the native path exists. #}
